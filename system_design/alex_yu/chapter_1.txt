  Request Cycle
    Users access servers using domain names
      DNS is a paid service provided by a 3rd party, and not hosted on our servers

    IP address is returned to the browser/ mobile app

    HTTP requests are sent to the IP adress

    The server returns HTML pages or JSON response for rendering 

Database
  with the growth of the user base, one server is not enough, we need multiple servers, one for web traffic, one for database

  seperating web and database servers allows themto be scaled independently

  [ user (web app, mobile app) ] <-> [ dns ]
     |
     |
     ↓
  [ Web Server ] <--------> [ Database ] 
               read/write/update

  which database to use,
    relational, non-relational

    relational databases aka SQL databases
      popular ones are MySQL, Oracle, PostgreSQL
      they store data in tables and rows
      you can perform join operations using SQL across different tables 

    non-relational databases aka NoSQL
      poplar ones are CouchDB, Neo4j, Cassandra, HBase, Amazon DynamoDB
      these databases are grouped into 4 categories 
        key-value stores
        graph stores
        column stores 
        document stores  
      join operations are generally not supported in non-relational databases

    generally relational databases are the best option
    non-relational databases might be the right choice if 
      > your application requires super-low latency
      > your data is unstructured, or if you do not have any relational data
      > you only need to serialize and deserialize data (JSON, XML, YAML)
          serialization -> converting an object in memory into a format that can be sent over the network
          deserialization -> converting the network format back to be stored as an object in memory
      > you need to store massive amounts of data
