reflection is a mechanism that allows a program to inspect and manipulate its own structure and behaviour at runtime

we use reflection for dynamic type inspection

reflection is also used in generic programming where we implement functions and data structures that operate on any type
reflection is used for serialization and deserialization 
used by ORM libraries
it is slower compared to direct code
reflection bypasses static type checks which can cause runtime errors

few important methods 
  reflect.TypeOf
  reflect.Value
  reflect.ValueOf
  reflect.ValueOf().Elem()


```go

    x := 42

    v := reflect.ValueOf(x)
    t := v.Type() // we cannot do x.Type()
                  // because int Tyoe doesn't have a method .Type()

    fmt.Println("type: ", t)  // represents the type of x  // int

    //   example to understand difference between kind and type

    fmt.Println("Is Int", t.Kind() == reflect.Int) // true

    }
```
