unbuffered channel
  `greeting := make(chan string)`
  this is an ubuffered channel

  as soon as you send data into an unbuffered channel it immediately blocks and waits for a receiver
    now if all other go routines terminate, or are also in hold, the go runtime realizes that there is a deadlock and throws an error

  // covered in channels.txt

buffered channel
  it is a channel with an associated storage
  it allows channels to hold a limited number of values before blocking the sender
    thus we can manage the rate of data transfer

  thus buffered channels allow asynchronous communication
  thus senders can continue working without blocking until buffer is full

  buffered channels also help with load balancing, handling bursts of data without immediate synchronization


  buffered channels 
    block on send
      only if the buffer is full

    block on receive 
      only if the buffer is empty
