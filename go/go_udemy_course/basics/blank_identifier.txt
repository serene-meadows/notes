-- blank identifier 
--
  `_` 
  it is a special identifier 
  you can assign anything to it, but never read from

  in go, 
    we get an error 
      if a variable is declared and not used
      if a package is imported and not used

  it is used to ignore a value

-- use cases
-- 
  function returns
    functions can return multiple values
    `_` is used to receive returns that are not going to be used in the code
    go compiler doesn't complain for `_` not being used
      ```go
        func swap(a, b int) (int, int) {
          return b, a
        }
        a, _ := swap(1, 3)
        _, b := swap(2, 4)
      ```
    don't use it to ignore `error` returns, always check errors 

  function parameters
    while implementing interfaces, 
    if you don't need the parameters, you can use the blank identifier to
    ignore them
    `func name(_ int, _ string) {}`

  range gives index, value
    you can use blank identifier if you don't need one of the values
    `for i, _ := range arrayName {}`


  importing a package for side effects
    if you want to import a package only for side effects 
    and not for direct use
    `import _ "net/http/pprof`
