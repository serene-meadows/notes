slices do not have a fixed length

slices are references to underlying arrays
they do not store any data themselves but provide a window into the array's elements

slices can grow and shrink dynamically


declaration
  var sliceName []elementType // zero initialized to nil

  var sliceName []int = []int{1, 2, 3}
  var sliceName = []int{1, 2, 3}
  sliceName := []int{1, 2}

  // initialize a slice using make()
  sliceName := make([]int, 5) // slice with len -> 5, cap -> 5 (defaults to len)
  sliceName := make([]int, 5, 10) // slice with len -> 5, cap -> 10

    // what are len and cap
    len -> number of elements currently in the slice
    cap -> number of elements in the underlying array starting from the slice's first element
           represents how much the slice can grow before it needs to allocate new memory


  // slicing an array
    when you slice an array, you are creting a new slice header that points to a specific
    segment of that array

    a := [5]int{1, 2, 3, 4, 5}
    sliceName := a[startIndex:endIndex] // [startIndex, endIndex)
      // array owns the memory, the slice is juts a "window" into that memory 
      ```golang
        arr := [5]int{1, 2, 3, 4, 5}
        s := arr[1:3] // [2, 3]
      ```
        Lenght -> 2 
        Capacity -> 4 (from index 1 to end of array)

        since array owns the memory, if you change s[0], arr[1] also changes

  // slicing a slice 
    when you slice a slice, you are creating a new slice header that points to the same 
    underlying array that the first slice was using 
    
    both slices not point to the same original background array 

    the new slice's capacity is limited by the capcity of the original slice, starting from 
    the new starting point 
      ```golang
        originalSlice := make([]int, 3, 10) // len 3, cap 10
        newSlice := originalSlice[1:3] // len 2, cap 9
      ```
