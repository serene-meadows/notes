-- struct tags
--
  A tag may be bound to a struct field when the field is declared
  Field tags are optional, the default value of each field tag is a blank string.

  Go allows both interpretted strings and raw string literals
  in practice, raw string literals are more used, because
  interpretted strings require escape sequences for many characters
    which is an issue since tags are generally used to represent key:"value" pairs
    with values enclosed in double quotes


  ```go
    struct {
      Title  string `json:"title" myfmt:"s1"`
      Author string `json:"author,omitempty" myfmt:"s2"`
      Pages  int    `json:"pages,omitempty" myfmt:"n1"`

      // although allowed
      // not good practice
      X, Y   bool   `myfmt:"b1"`

    }

  ```


  -- reflection
  --
    reflection is used to inspect field tag information






-- purpose of field tags
--
  The purpose of each field tag is application dependent.
  example
    ```go
      struct {
        Title  string `json:"title" myfmt:"s1"`
        Author string `json:"author,omitempty" myfmt:"s2"`
        Pages  int    `json:"pages,omitempty" myfmt:"n1"`
      }

    ```

    encoding/json standard package uses field tags to determine field names in JSON texts, in the process of encoding
    struct values into JSON texts or decoding JSON texts into struct values

    The functions in the encoding/json standard package will only encode and decode the exported struct fields
    which is why the first letters of the field names in the above example are all upper cased.





-- limitations
--
  unlike c, go structs don't support union





-- exported fields
--
  Only exported fields of struct types shown up in a package can be used in other packages by importing the package
  We can view non-exported struct fields as private/protected member variables.





-- order of fields and field tags
--
  The field tags and the order of the field declarations in a struct type matter for the identity of the struct type.
  Two unnamed struct types are identical only if they have the same sequence of field declarations
  Two field declarations are identical only if
    their respective names, their respective types and their respective tags are all identical





-- no recusrive reference
--
  A struct type can't have a field of the struct type itself, neither directly nor recursively.
