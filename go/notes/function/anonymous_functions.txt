-- anonymous functions
--
  The definition of an anonymous function is almost the same as a function declaration
  except there is no function name portion in the anonymous function definition.

  An anonymous function can be called right after it is defined.

  you can call anonymous functions within functions, anonymous functions
    works because functions are 
      functions as types (i.e. functions are first class citizen/ first class objects)
        // first class citizens ca 
        // > be assigned to a variable 
        // > be passed as an argument 
        // > be return from a function 
        // > be stored in data structures

    only restriction is that named functions cannot be defined inside another function declaration

  example - calling anonymous functions directly
    ```go
      package main

      func main() {
        // This anonymous function has no parameters
        // but has two results.
        x, y := func() (int, int) {
          println("This function has no parameters.")
          return 3, 4
        }() // Call it. No arguments are needed.

        // The following anonymous function have no results.

        func(a, b int) {
          // The following line prints: a*a + b*b = 25
          println("a*a + b*b =", a*a + b*b)
        }(x, y) // pass argument x and y to parameter a and b.

        func(x int) {
          // The parameter x shadows the outer x.
          // The following line prints: x*x + y*y = 32
          println("x*x + y*y =", x*x + y*y)
        }(y) // pass argument y to parameter x.

        func() {
          // The following line prints: x*x + y*y = 25
          println("x*x + y*y =", x*x + y*y)
        }() // no arguments are needed.
      }
    ```
      // observation
      // in the last anonymous function
      // since the anonymous function is in the scope of the 'x' and 'y' variables
      // it can use the variables directly, without needing them to be passed
      // this is closure
      // The term Closure comes from the fact that the function "closes over" (or captures) the variables in its surrounding environment.
