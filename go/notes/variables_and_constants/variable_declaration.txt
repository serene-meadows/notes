-- variables
--
  variables are named values
  variables are stored in memory at run time
  the value represented by a variable can be modified at run time

  all variables are typed
  when declaring a variable there must be sufficient information provided
    for compilers to deduce the type of the variable

  -- where can variables be declared
  --
    variables declared within functions are called local variables
    and those declared at package scope are called package-level variables or global variables





-- variable declaration
--
  there are two basic types of variable declaration
    standard
    short 

  -- standard declaration
  --
    each standard variable declaration starts with the 'var' keyword
    example 
      // when multiple variables declared in one line, with specifying the type on lhs
      // the type has to be common
      ```go
        var lang, website string = "Go", "https://golang.org"
        var compiled, dynamic bool = true, false 
        var announceYear int = 2009
        
      ```


    -- standard declaration without specifying type - implicit conversion
    --
      Compilers will deduce the types of the declared variables as the types (or default types) of their respective initial values. 

      // when multiple variables declared in one line, without specifying the type on lhs
      // the type has doesn't have to be common
      ```go
        // The types of the lang and dynamic variables
        // will be deduced as built-in types "string"
        // and "bool" by compilers, respectively.
        var lang, dynamic = "Go", false

        // The types of the compiled and announceYear
        // variables will be deduced as built-in
        // types "bool" and "int", respectively.
        var compiled, announceYear = true, 2009

        // The types of the website variable will be
        // deduced as the built-in type "string".
        var website = "https://golang.org"

      ```


    -- standard declaration without specifying value
    -- 
      in these declarations, all declared variables are initialized zero values of their respective types

      example
      ```go
        // Both are initialized as blank strings.
        var lang, website string

        // Both are initialized as false.
        var interpreted, dynamic bool

        // n is initialized as 0.
        var n int

      ```

    -- standard declaration - variable block
      ```go
        var (
          lang, bornYear, compiled     = "Go", 2007, true
          announceAt, releaseAt    int = 2009, 2012
          createdBy, website       string
        )

      ```


    -- semantics
    -- 
      -- pure assignment
      --
        In the above variable declarations, the sign = means assignment
        Once a variable is declared, we can modify its value by using pure value assignments.

        The expression items at the left of = symbol in a pure assignment are called destination or target values.
        the destination must be 
          addressable values, map index expressions, or the blank identifier. 

        the reason constants don't have to be addressable, is because since they are immutable
        they won't appear on the lhs of a pure assignment

        example 
          ```go
            const N = 123
            var x int
            var y, z float32

            N = 9 // error: constant N is not modifiable
            y = N // ok: N is deduced as a float32 value
            x = y // error: type mismatch
            x = N // ok: N is deduced as an int value
            y = x // error: type mismatch
            z = y // ok
            _ = y // ok

            x, y = y, x // error: type mismatch
            x, y = int(y), float32(x) // ok
            z, y = y, z               // ok
            _, y = y, z               // ok
            z, _ = y, z               // ok
            _, _ = y, z               // ok
            x, y = 69, 1.23           // ok

          ```

        Go doesn't support assignment chain.
        example - ERR
          ```go
            var a, b int
            a = b = 123 // syntax error

          ```


  -- short declaration
  -- 
    We can also use short variable declaration forms to declare variables.
    Short variable declarations can only be used to declare local variables.

    example 
      ```go
        package main

        func main() {
          // Both lang and year are newly declared.
          lang, year := "Go language", 2007

          // Only createdBy is a new declared variable.
          // The year variable has already been
          // declared before, so here its value is just
          // modified, or we can say it is redeclared.
          year, createdBy := 2009, "Google Research"

          // This is a pure assignment.
          lang, year = "Go", 2012

          print(lang, " is created by ", createdBy)
          println(", and released at year", year)
        } 

      ```

      
    In short form declaration
      the var keyword and variable types must be omitted

      The assignment sign must be := instead of =


      Each short variable declaration must declare at least one new variable.
      old variables and new variables can mix at the left of :=. But there must be at least one new variable at the left.





-- each local declared variable must be used at least once effectively
--
  go compiler doesn't allow local variables declared but not used
  pckage-level variables have no such condition

  example - ERR 
    ```go
      package main

      // Some package-level variables.
      var x, y, z = 123, true, "foo"

      func main() {
        var q, r = 789, false
        r, s := true, "bar" // ERR
                            // s is unused
                            
        r = y // ERR
              // r is unused.
              
        x = q // q is used.
      }
    ```
        // in such cases, hacky fix is to temporarily use them to assign value to blank identifier
        ```go
          _, _ = r, s // counts as r and s being used
        ```





-- Dependency relations of package-Level variables affect their initialization order
--
  example 
    ```go
      var x, y = a+1, 5         // 8 5
      var a, b, c = b+1, c+1, y // 7 6 5
    ```
      // the initialization order of the package-level variables are y = 5, c = y, b = c+1, a = b+1, and x = a+1.

  the only codition is that they packet-level pacakges cannot be circularly dependent
  example - ERR 
    ```go
      var x, y = y, x 
    ```





-- addressable
--
  all variables are addressable
  all constants are unaddressable
