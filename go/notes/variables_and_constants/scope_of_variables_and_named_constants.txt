-- scope of variables
-- and named constants
--
  we can use a pair of { and } to form a code block.
  A code block can nest other code blocks.

  A variable or a named constant declared in an inner code block wil
  shadow the variables and constants declared with the same name in outer code blocks

  example 
    ```go
      package main

      const y = 789
      var x int = 123

      func main() {
        // The x variable shadows the above declared
        // package-level variable x.
        var x = true

        // A nested code block.
        {
          // Here, the left x and y are both
          // new declared variable. The right
          // ones are declared in outer blocks.
          x, y := x, y

          // In this code block, the just new
          // declared x and y shadow the outer
          // declared same-name identifiers.
          x, z := !x, y/10 // only z is new declared
          y /= 100
          println(x, y, z) // false 7 78
        }

        println(x) // true
        println(z) // error: z is undefined.
      }

    ```

  -- scope of a package level named constant / variable
  --
    scope is the whole package the named constant/ variable is declared in 
    
  -- scope of a local named constant/ variable 
  --
    begins at the end of its declaration and ends at the end of its innermost containing code block
