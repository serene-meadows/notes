-- Return Pointers of Local Variables Is Safe in Go
--
  Unlike C language, Go is a language supporting garbage collection
  so return the address of a local variable is absolutely safe in Go.

  example
    ```go
      func newInt() *int {
        a := 3
        return &a
      }

    ```

  this program would have led to a crash in c, because the memory for 'a' would be deleted the moment the function finished
  (stack memort)
  so the pointer would have been pointing to freed memory

  where as in go, since it garbage collected
  it does something called escape analysis
    The Go compiler looks at your code and asks: "Does this variable 'escape' the function?"

    If the variable stays inside the function:
      It is stored on the Stack. This is very fast and automatically cleaned up when the function ends.

    If the variable is returned as a pointer:
      It "escapes" the function. The compiler automatically moves that variable to the Heap.

  The memory stays alive as long as someone is still pointing to it.
  The Garbage Collector (GC) will only delete it once it's no longer being used.

  thus you don't have to worry about dangling pointers when a pointer is returned from a function
