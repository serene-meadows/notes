-- untyped and typed values
--
  An untyped value means the type of the value has not been confirmed yet
  A typed value means that the type of the value is determined

  For most untyped values, each of them has one default type.
  The predeclared nil is the only untyped value which has no default type.

  the basic type literals are all called unnamed constants (or literal constants)
    except 'false' and 'true'
    'false' and 'true' are built-in named constants

  All literal constants are untyped values

  The default type of a literal constant is determined by its literal form.
    The default type of 
      a string literal is         string
      a boolean literal is        bool
      an integer literal is       int
      a rune literal is           rune (a.k.a., int32)
      a floating-point literal is float64





-- explicit conversion of untyped constants
--
  -- Type conversion 
  --
    T(v) - convert a value v to the type T (if conversion is legal)
    sometimes, the form of explicit conversions must be written as (T)(v) to avoid ambiguities.

  
  -- Rules for explicit type conversion for literal constants
  --
    For an untyped constant v, for T(v) to be legal
      Rule 1
        v (or the literal denoted by v) is representable as a value of a basic type T. 
        The result value is a typed constant of type T.

      Rule 2
        The default type of v is an integer type (int or rune) and T is a string type
        The result of T(v) is a string of type T and contains the UTF-8 representation of the integer as a Unicode code point.

        Integer values outside the range of valid Unicode code points result strings represented by "\uFFFD" (a.k.a., "\xef\xbf\xbd")
        0xFFFD is the code point for the Unicode replacement character.

        The result string of a conversion from an integer always contains one and only one rune.

    examples - OK
      ```go
        // Rounding happens in the following 3 lines.
        complex128(1 + -1e-1000i)  // 1.0+0.0i
        float32(0.49999999)        // 0.5
        float32(17000000000000000)
        // No rounding in the these lines.
        float32(123)
        uint(1.0)
        int8(-123)
        int16(6+0i)
        complex128(789)

        string(65)          // "A"
        string('A')         // "A"
        string('\u68ee')    // "森"
        string(-1)          // "\uFFFD"
        string(0xFFFD)      // "\uFFFD"
        string(0x2FFFFFFFF) // "\uFFFD" 

      ```
  examples - ERR
    ```go
      // 1.23 is not representable as a value of int.
      int(1.23)
      // -1 is not representable as a value of uint8.
      uint8(-1)
      // 1+2i is not representable as a value of float64.
      float64(1+2i)

      // Constant -1e+1000 overflows float64.
      float64(-1e1000)
      // Constant 0x10000000000000000 overflows int.
      int(0x10000000000000000)

      // The default type of 65.0 is float64,
      // which is not an integer type.
      string(65.0)
      // The default type of 66+0i is complex128,
      // which is not an integer type.
      string(66+0i)

    ```





-- Type Deductions in Go
-- 
  Type deduction is also often called type inference.
  It means that when possible, you don't need to explicitly the type of a value
    go's compiler will deduce the type by context

  if a place needs a value of a certain type and an untyped value (often a constant)
    is representable as a value of the certain type
  Go compilers will view the untyped value as a typed value of the certain type

    this is commonly done with
      operands in an operator operation 
      argument in a function call
      destination value in an assignment
      source value in an assignment
    
  if circumstance is such that there is no requirement on the type of the untyped value
    go compilers will treat the untyped value as a typed value of it's default type


  example - tricky error 
    ```go
      // error: -1 overflows uint
      const MaxUint_b uint = ^0 
      // the reason ^0 gives -1, even though the consxt is uint(...)
      // its because the default type of a number is int, and since ^0 is an operation
      // it will use the default value of int (signed) which leads to -1, which not convertible
      // to uint
    ```





-- Explicit Conversions on Non-Constant Numeric Values
-- 
  in go, two typed values of two different basic types cannot be assigned to each other
  i.e. the type of the source and destination values in an assignemnt must be identical if the 
  two values are both basic values

  in the case that the type of the source and destination value being different
  then source value must be explicitly converted to the type of destination value

  in addition to above case,
    non constant floating point and integer values can be explicitly converted to any other 
    floating point and integer types

    non constant complex values can be explicitly converted to any other complex type

  
  unlike constant number conversions, overflows are allowed in non constant number conversions
  and when converting a non constant floating point value to an integer, rounding is allowed (towards zero)

  example 
    ```go
      const a = -1.23

      // The type of b is deduced as float64.
      var b = a

      // error: constant 1.23 truncated to integer.
      var x = int32(a)

      // error: cannot assign float64 to int32.
      var y int32 = b

      // okay: z == -1, and the type of z is int32.
      //       The fraction part of b is discarded.
      var z = int32(b)

      const k int16 = 255

      // The type of n is deduced as int16.
      var n = k

      // error: constant 256 overflows uint8.
      var f = uint8(k + 1)

      // error: cannot assign int16 to uint8.
      var g uint8 = n + 1

      // okay: h == 0, and the type of h is uint8.
      //       n+1 overflows uint8 and is truncated.
      var h = uint8(n + 1)

    ```





-- value denoted by an untyped constant can overflow its default type
--
  ```go
    // 3 untyped named constants. Their bound
    // values all overflow their respective
    // default types. This is allowed.
    const n = 1 << 64          // overflows int
    const r = 'a' + 0x7FFFFFFF // overflows rune
    const x = 2e+308           // overflows float64

    func main() {
      _ = n >> 2
      _ = r - 0x7FFFFFFF
      _ = x / 2
    }
  ``` 





-- value denoted by an typed constant cannot overflow its type
--
  ```go - ERR
    // 3 typed named constants. Their bound
    // values are not allowed to overflow their
    // respective default types. The 3 lines
    // all fail to compile.
    const n int = 1 << 64           // overflows int
    const r rune = 'a' + 0x7FFFFFFF // overflows rune
    const x float64 = 2e+308        // overflows float64
  ```





-- constant identifiers are replaced by literal value at compile time
--
  Constant declarations can be viewed as enhanced #define macros in C

  All the occurrences of a named constant will be replaced with the literal it represents at compile time.

  If the two operands of an operator operation are both constants, then the operation will be evaluated at compile time

  example 
    // at compile time the following code
    ```go
      package main

      const X = 3
      const Y = X + X
      var a = X

      func main() {
        b := Y
        println(a, b, X, Y)
      }
    ```
    // will be viewed as 
    ```go
      package main

      var a = 3

      func main() {
        b := 6
        println(a, b, 3, 6)
      }
    ```
