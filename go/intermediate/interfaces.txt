interfaces provide a way to specify behaviours 
  they define a set of method signatures that a type must implement to satisfy the interface 

  they help with code reuse, decoupling, polymorphism without relying on inheritance (go does not support inheritance)

  in go, interface implementation is implicit, (there is no implements keyword)
  if a type's method set contains all the methods declared by an interface, the type implements that interface automatically 



  interface in go, is a way to define a set of methods that other types must implement in order for them to be considered

  // implementing interface - (taking geometry as example)
    a type implicitly satisfies an interface if it implements all the methods defined by that interface 

    ````go
      // types 
        // geometry interface declared above 

        type rect struct {
          width, height float64
        }

        type circle struct {
          radius float64
        }


      // implementing methods 
        // rect
          func(r rect) area() {
            return r.width * r.height
          }

          func (r rect) permim() float64 {
            return 2 * (r.height + r.width)
          }

        // circle
          func(c circle) area() {
            return math.Pi * c.radius * c.radius
          }

          func (c circle) perim() float64 {
            return 2 * math.Pi * c.radius 
          }

          func (c circle) diameter() float63 {
            return 2 * c.radius
          }
    ````
