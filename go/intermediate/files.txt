the os package in go, provides functions for operating system functionally,
  including file operations 
  prefer buffered reader and writers

key components 
  Create(name string) (*File, error)
  OpenFile(name string, flag int, perm FileMode) (*File, error)
  Write(b []byte) (n int, err error)
  WriteString(s string) (n int, err error)

  // Create 
    // Create or Truncate a file with a given name 
    // returns a file descriptor of the created file 

  // OpenFile
    // open a file with the given name flags and permissions 
    // returns a file descriptor

  // os.File implements the write interface, so it can be used as a writer


example - write
  ````go
    file, err := os.Create("output.txt")
    if err != nil {
      fmt.Println(err)
      return
    }
    defer file.Close()


    // write data to file - byte slice
    data := []byte("Hello world \n")
    _, err = file.Write(data)
    if err != nil {
      fmt.Println(err)
      return
    }

    // write data to file - string 
    file, err = file.Create("output2.txt")
    if err != nil {
      fmt.Println(err)
      return
    }
    defer file.Close()
    
    _, err = file.WriteString("Hello worl")
    if err != nil {
      return
    }
  ````

reading - read 
  ````go
    file, err := os.Open("input.txt") 
    if err != nil {
      return
    }
    defer func() {
      fmt.Println("closing file")
      file.Close()
    }()
    
    // read the contents of the open file 
    data := make([]byte, 1024) // buffer to read file into
    _, err = file.Read(data)
    if err != nil {
      return
    }
    
    fmt.Println(string(data))
      // outputs the contents of the whole file 

    // using buffered reader to read line by line 
    scanner := bufio.NewScanner(file)

    for scanner.Scan() {
      line := scanner.Text()
        // .Text() returns the most recent tokens generated by a call to the scanner.Scan() as a newly allocated string
        // with every scanner.Scan() it keeps overriding the string(token) it returns
      fmt.Println(line)
    }
    err = scanner.Err()
    if err != nil {
      fmt.Println("error reading file", err)
    }
    // scanner.Scan() returns bool value, err is kept seperate in scanner.Err() and retrieved by calling after the loop
      // it returns false, when it 
        > reaches EOF 
        > error case (in that case it saves error internally, and error can be accessed via scanner.Err())
    // scanner.Err()
      > in case of EOF it returns nil
      > in case of err case, it returns error
  ````

EOF
  End of File 
    Whenever a file is created, there is something at the end, and the moment we reach the end of file,
    we will see the end of file character.

    In older systems, EOF used to be denoted by some special character 

    EOF here, is denoted by the absence of data 
    that's why when printing the lines, the last line (which is empty) is not printed because there is no data after it 
      the previous empty lines will be printed, because they still have data after them (more empty lines) that its not counted as EOF
