pointer
  a pointer is a variable that stores the memory address of another variable 
  pointer variables have their specific types 
    a pointer could be an integer pointer, floating point pointer, string pointer, so on...

use cases
  modify the value of a variable indirectly
  we can pass large data structures efficiently between functions using pointers 
  manage memory directly for performance reasons

pointer operations 
  limited to referencing (&) and dereferencing (*)

  go doesn't support pointer arithmetic like c/ c++
  

declaration
  var ptr *int // zero-value -> nil

  var a int = 1
  ptr = &a  // ptr now points to a's memory address
      // &var -> gives the address of the var in RAM

  fmt.Println(*ptr) // dereference - prints the value at the memory address
  fmt.Println(ptr) // prints the memory address

passing pointer to a function
  whenever you pass a variable to a function, a copy of that variable is passed 
  thus to modify the variable, you must pass the memory address of that variable
  ```go
    func modifyValue(ptr *int) {
      // increment the value at the ptr
      *ptr++
    }

    a := 10
    ptr := &a
    modifyValue(ptr)
    fmt.Println(a) // 11
  ```

go also has a unsafe pointer 
  Go normally guarantees:
    Type safety
    Memory safety
    Garbage collector protection
    No pointer arithmetic

  Using unsafe lets you:
    Perform pointer arithmetic
    Reinterpret memory as another type
    Break type safety
    Access private struct fields
    Potentially corrupt memory

  If misused, it can cause:
    Crashes
    Data corruption
    Subtle bugs
    Security vulnerabilities
