const -> immutable 

syntax for declaring const 
  // instead of var -> const 
  const tuesday = 2      // untyped const
  const thursday int = 4 // typed const

  multiple const variables declaration 
  // const block
  const (
    monday = 1
    tuesday = 2      // untyped const
    thursday int = 4 // typed const
  )

  canot use := notation to declare a const
  
typed vs untyped const 
  untyped const 
    dont have a fixed type until used 

    example 
      const x = 10

      var i int = x 
      var f float64 = x


const have to be initialized with values that can be determined at compile time 

zero initialized
  const have to be initialized so zero-initialization doesn't apply


memory
  Do NOT occupy memory
  Exist only at compile time

  when you write 
    const x = 10 

  Go does not create a memory location for x.

  There is no address.
  You cannot take &x.
  It is not stored in RAM like a variable.

  Instead, the compiler literally replaces x with 10 everywhere it’s used.

  Think of it like a text replacement during compilation.
