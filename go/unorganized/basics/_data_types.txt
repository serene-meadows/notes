type keyword
  // using string and boolean as example, but can apply to other types as well
  we can define custom boolean and string types for the built-in boolean and string types.
  although there is only one built-in type for each of boolean and string types, 
  So there can be many boolean and string types. 

  ```go
    // status and bool are two different types.
    type status bool

    // MyString and string are two different types.
    type MyString string

    // Id and uint64 are two different types.
    type Id uint64

    // real and float32 are two different types.
    type real float32


    /* Some type alias declarations */

    // boolean and bool denote the same type.
    type boolean = bool

    // Text and string denote the same type.
    type Text = string

    // U8, uint8 and byte denote the same type.
    type U8 = uint8

    // char, rune and int32 denote the same type.
    type char = rune
  ```
  








default/ zero values 
  Each type has a zero value. 
  The zero value of a type can be viewed as the default value of the type.

In Go, two typed values of two different basic types can't be assigned to each other.
the types of the destination and source values in an assignment must be identical if the two values are both basic values
If the type of the source basic value is not same as the type of the destination basic value,
  the source value must be explicitly converted to the type of the destination value.
