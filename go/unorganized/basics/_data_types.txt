type keyword
  // using string and boolean as example, but can apply to other types as well
  we can define custom boolean and string types for the built-in boolean and string types.
  although there is only one built-in type for each of boolean and string types, 
  So there can be many boolean and string types. 

  ```go
    // status and bool are two different types.
    type status bool

    // MyString and string are two different types.
    type MyString string

    // Id and uint64 are two different types.
    type Id uint64

    // real and float32 are two different types.
    type real float32


    /* Some type alias declarations */

    // boolean and bool denote the same type.
    type boolean = bool

    // Text and string denote the same type.
    type Text = string

    // U8, uint8 and byte denote the same type.
    type U8 = uint8

    // char, rune and int32 denote the same type.
    type char = rune
  ```

assignable meaning
  if x = y is legal
  we can say type y is assignable to type x
    that means
      type y is identical to type x
      type y can be implicitly converted to type x
      x is the blank identifier






// Go does NOT do implicit type conversion.
// you need to do it explicitly
// If types differ → you must convert explicitly.
//   this doesn't apply to un typed consts because well they are "untyped" until they are in a scenario which requires a type
//   and it implicitly converts to the expected type, it should still be similar tho, cannot assign numeric unsigned typed const to a string
//   however for typed consts, you need to explicitly ocnvert to another type if need so
  ```go
    // In Go, values of different basic types cannot be assigned to each other.
      var a int = 10
      var b float64 = 20.5
      a = b // error

    // You must explicitly convert the source value to the destination type.
      var a int
      var b float64 = 20.5

      a = int(b) // OK

    // The same rule applies to operations between different types.
    var x int = 10
    var y float64 = 5.5

    z := x + y // error 

    // Convert one value so both operands have the same type.

    var x int = 10
    var y float64 = 5.5

    z := float64(x) + y // OK

    // tldr
    
    // Key rule in Go:
    // Basic types must match for:
      // 1. Assignments
      // 2. Arithmetic operations
      // 3. Comparisons
  ```
