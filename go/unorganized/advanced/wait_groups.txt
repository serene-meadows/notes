a wait group is a synchronization primitive provided by the sync package in go
  it is used to wait for a collection of go routines to complete their execution 

Add(delta int)
  increments the wait group counter by the argument that we pass it 

Done()
  decrements the counter by one 

Wait()
  blocks until the counter inside the wait group reaches zero

```go
  type Worker struct {
    ID int
    Task string
  }

  func (w *Worker) PerformTask(wg *sync.WaitGroup) {
    defer wg.Done()

    fmt.Printf("Worker ID %d started %s\n", w.ID, w.Task)
    time.Sleep(time.Second)
    fmt.Printf("Worker ID %d finished %s\n", w.ID, w.Task)
  }

  func main() {
    var wg sync.WaitGroup

    tasks := []string {"process_0", "process_1", "process_2"}

    for i, task := range tasks {
      worker := Worker{ID: i + 1, Task: task}
      wg.Add(1)
      go worker.PerformTask(&wg)
    }

    wg.Wait()

    fmt.Println("Done fam its done")
  }
```





best practices  
  avoid blocking calls inside go routines
    if they are blocked for ever -> wait groups block forever

  use defer to call Done()

  ensure that you don't use Add(1) inside the go routine

  don't overcount wg.Add
    if wg.Done is less -> deadlock
    if wg.Done is more -> some go routines won't be waited for
