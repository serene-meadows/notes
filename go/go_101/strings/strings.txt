In Go, a string is a fundamental, built-in type. While it behaves like a basic
value, under the hood it is actually a composite data structure—specifically,
an immutable sequence of bytes.





-- Internal Structure and Memory Sharing
--
  For the standard Go compiler, a string is internally represented as a 
  two-word (16-byte on 64-bit systems) descriptor or "header":
    ```go
      type _string struct {
        elements *byte // Pointer to the underlying array of bytes
        len      int   // The number of bytes (not necessarily characters)
      }

    ```

  -- Immutability
  --
    The contents (the underlying bytes) of a string are STRICTLY IMMUTABLE.
    - You cannot change a specific byte inside a string.
    - You cannot change the length of an existing string value.
    - You CAN reassign a string variable entirely to point to a new string.


  -- Memory Sharing (Zero-Cost Substrings)
  --
    Because strings are immutable, it is perfectly safe for multiple strings 
    to point to the exact same underlying byte array.
    When you assign a string to another variable, or take a substring, Go ONLY
    copies the 16-byte header. No underlying data is copied.

  
  example
    ```go
      var helloWorld = "hello world!"


      // `hello` is a substring. It shares the exact same underlying memory 
      // as `helloWorld`. It just has a different length in its header.
      var hello = helloWorld[:5] 


      // Indexing a string retrieves the byte (`uint8`) at that position.
      // 104 is the ASCII/UTF-8 decimal value for 'h'.
      fmt.Printf("Value: %d, Type: %T\n", hello[0], hello[0]) // Value: 104, Type: uint8


      // -- IMMUTABILITY ENFORCEMENT --
      // The elements of a string are unaddressable and immutable.
      // hello[0] = 'H'         // COMPILE ERROR: cannot assign to hello[0]
      // fmt.Println(&hello[0]) // COMPILE ERROR: cannot take the address of hello[0]


      // The `strings` package provides functions for string manipulation, 
      // since the built-in type has no methods.
      fmt.Println(strings.HasPrefix(helloWorld, hello)) // true

    ```





-- Literals and Basic Properties
--
  -- Literal Representations
  --
    1. Interpreted Strings (Double Quotes `""`): Support escape sequences like `\n`.
    2. Raw Strings (Backticks ``): Treat all characters literally. Can span multiple lines.

  
  -- Properties
  --
    - Zero Value: `""` (the empty string).
    - Concatenation: Performed using `+` or `+=`.
    - Comparison: Strings are comparable using `==`, `!=`, `<`, `>`, `<=`, `>=`.
      Comparison is done byte-by-byte (lexicographically). If one string is a
      prefix of another, the longer string is considered "larger".






-- The Constant Evaluation Trap
--
  If a string is a constant, `len(s)` is evaluated at COMPILE TIME and becomes
  a constant expression.
  However, a substring operation `s[:]` is NOT a constant expression, so
  `len(s[:])` evaluates at RUNTIME.


  ```go
    const s = "Go101.org" // len(s) == 9 (Constant)


    // Since len(s) is a constant, this shift operation happens at compile time
    // using high-precision math.
    // 1 << 9 = 512. Then 512 / 128 = 4.
    var a byte = 1 << len(s) / 128


    // Since len(s[:]) is NOT a constant, this evaluates at runtime.
    // `1` defaults to `int`. 1 << 9 = 512. 
    // BUT, we are trying to fit this into a `byte` (uint8) before division!
    // 512 overflows a byte and wraps to 0. Then 0 / 128 = 0.
    var b byte = 1 << len(s[:]) / 128


    fmt.Println(a, b) // Output: 4 0

  ```





-- Encoding: Bytes, Runes, and UTF-8
--
  Go source code and string literals are strictly UTF-8 encoded.

  Byte: An alias for `uint8`. Represents a raw piece of data.
  Rune: An alias for `int32`. Represents a single Unicode Code Point 
  (a distinct character or symbol).


  In UTF-8, a single rune can take anywhere from 1 to 4 bytes.
  - English characters typically take 1 byte.
  - Eastern characters (like Chinese) typically take 3 bytes.
  - Emojis typically take 4 bytes.


  CRITICAL DISTINCTION: `len()` returns the number of BYTES, not the 
  number of RUNES (characters).


  example
    ```go
      // "Go" takes 2 bytes. "世界" (World) takes 6 bytes (3 bytes each).
      s := "Go世界"

      fmt.Printf("Number of BYTES: %d\n", len(s)) // Output: 8
      
      // To get the actual character count, use the utf8 package.
      fmt.Printf("Number of RUNES: %d\n", utf8.RuneCountInString(s)) // Output: 4

    ```





-- String Conversions
--
  Because strings are immutable, converting them to mutable slices (and vice versa)
  generally requires allocating new memory and copying the data.

  
  example
    ```go
      // String <-> Byte Slice ([]byte) 
      // Used for raw binary manipulation.
      // This allocates a new byte slice and COPIES the bytes to prevent the 
      // slice from mutating the original immutable string.
      byteSlice := []byte(str)
      byteSlice[0] = 'H' // Mutating the slice is safe.
      

      newStr := string(byteSlice)
      fmt.Println(newStr) // Output: "Hello"


      // String <-> Rune Slice ([]rune) 
      // Used when you need to manipulate individual Unicode characters.
      // This decodes the UTF-8 string into an array of int32 code points.
      

      chineseStr := "世界"
      runeSlice :=[]rune(chineseStr)
      

      fmt.Printf("First rune: %c\n", runeSlice[0]) // Output: 世
      

      // Convert back to string (encodes the int32s back into UTF-8 bytes)
      backToStr := string(runeSlice)
      _ = backToStr

    ```





-- String and Rune/Byte Conversions (The Deep Copy Rule)
--
  Because strings are immutable and slices are mutable, converting between them
  ALWAYS requires allocating new memory and making a "deep copy" of the data.
  If they shared memory, you could mutate a string by modifying the slice!


  -- String <-> Byte Slice (`[]byte`)
  --
    - `[]byte(s)`: Copies the exact raw bytes of the string. Illegal UTF-8 bytes
      are copied exactly as they are without modification.
    - `string(bs)`: Copies the bytes into a new immutable string structure.

  
  -- String <-> Rune Slice (`[]rune`)
  --
    - `[]rune(s)`: Decodes the string's UTF-8 bytes into Unicode Code Points (int32).
      If it encounters invalid UTF-8 bytes, it inserts the Unicode Replacement
      Character `0xFFFD` () into the rune slice.
    - `string(rs)`: Encodes the runes back into UTF-8 bytes. If a rune is outside
      the valid Unicode range, it is encoded as `0xFFFD` (which takes 3 bytes:
      0xEF, 0xBF, 0xBD).

    
  -- Byte Slice <-> Rune Slice
  --
    Go does not have a direct built-in syntax like `[]rune(byteSlice)`.

    Methods to achieve this:
      1. The Hop (Inefficient): `rs :=[]rune(string(bs))` (Requires two allocations).
      2. The `bytes` package: `rs := bytes.Runes(bs)`
      3. The `unicode/utf8` package: Manual looping and encoding/decoding.


    example
      ```go
        import (
          "bytes"
          "unicode/utf8"
        )


        func Runes2Bytes(rs []rune) []byte {
          n := 0
          for _, r := range rs {
            n += utf8.RuneLen(r)
          }
          n, bs := 0, make([]byte, n)
          for _, r := range rs {
            n += utf8.EncodeRune(bs[n:], r)
          }
          return bs
        }



        func main() {
          s := "Color Infection is a fun game."
          bs := []byte(s) // string -> []byte
          s = string(bs)  // []byte -> string
          rs := []rune(s) // string -> []rune
          s = string(rs)  // []rune -> string
          rs = bytes.Runes(bs) // []byte -> []rune
          bs = Runes2Bytes(rs) // []rune -> []byte
        }

      ```





-- Compiler Optimizations for Conversions Between Strings and Byte Slices
--
  -- for-range on Strings
  --
    The for-range loop control flow applies to strings. But please note, for-range will iterate
    the Unicode code points (as rune values),instead of bytes, in a string. Bad UTF-8 encoding
    representations in the string will be interpreted as rune value 0xFFFD.


    ```go
      s := "éक्षिaπ囧"
      for i, rn := range s {
        fmt.Printf("%2v: 0x%x %v \n", i, rn, string(rn))
      }
      fmt.Println(len(s))


      // output
      //  0: 0x65 e
      //  1: 0x301 ́
      //  3: 0x915 क
      //  6: 0x94d ्
      //  9: 0x937 ष
      // 12: 0x93f ि
      // 15: 0x61 a
      // 16: 0x3c0 π
      // 18: 0x56e7 囧
      // 21

    ```

      - the iteration index value may be not continuous. The reason is the index is the byte index
        in the ranged string and one code point may need more than one byte to represent.
      - the first character, é, is composed of two runes (3 bytes total)
      - the second character, क्षि, is composed of four runes (12 bytes total).
      - the English character, a, is composed of one rune (1 byte).
      - the character, π, is composed of one rune (2 bytes).
        the Chinese character, 囧, is composed of one rune (3 bytes).


  // to iterate through bytes 
    ```go
      s := "éक्षिaπ囧"
      for i := 0; i < len(s); i++ {
        fmt.Printf("The byte at index %v: 0x%x \n", i, s[i])
      }

    ```


  // we can also make use of the compiler optimization mentioned above to iterate bytes in a string.
    ```go
      s := "éक्षिaπ囧"
      // Here, the underlying bytes of s are not copied.
      for i, b := range []byte(s) {
        fmt.Printf("The byte at index %v: 0x%x \n", i, b)
      }

    ```


  From the above several examples, we know that len(s) will return the number of bytes in string s
  The time complexity of len(s) is O(1).
  to get the number of runes in a string
    - Using a for-range loop to iterate and count all runes is a way
    - and using the RuneCountInString function in the unicode/utf8 standard package is another way
    - The efficiencies of the two ways are almost the same
    - The third way is to use len([]rune(s)) to get the count of runes in string s
      the standard Go compiler makes an optimization for the third way to avoid an unnecessary deep copy so that it is as efficient as the former two ways
      the time complexities of these ways are all O(n).






-- Sugar: Use Strings as Byte Slices
--
  we can use the built-in copy and append functions to copy and append slice elements
  In fact, as a special case, if the first argument of a call to either of the two functions is a byte slice, then the second argument can be a string
  (if the call is an append call, then the string argument must be followed by three dots ...)
  In other words, a string can be used as a byte slice for the special case.

  ```go
      hello := []byte("Hello ")
      world := "world!"


      // The normal way:
      // helloWorld := append(hello, []byte(world)...)
      helloWorld := append(hello, world...) // sugar way
      fmt.Println(string(helloWorld))


      helloWorld2 := make([]byte, len(hello) + len(world))
      copy(helloWorld2, hello)
      // The normal way:
      // copy(helloWorld2[len(hello):], []byte(world))
      copy(helloWorld2[len(hello):], world) // sugar way
      fmt.Println(string(helloWorld2))

  ```






-- More About String Comparisons
-- 
  Go compilers will make the following optimizations for string comparisons.
    - For == and != comparisons, if the lengths of the compared two strings are not equal,
      then the two strings must be also not equal (no needs to compare their bytes).
    - If their underlying byte sequence pointers of the compared two strings are equal, then the
      comparison result is the same as comparing the lengths of the two strings.


  So for two equal strings, the time complexity of comparing them depends on whether or not
  their underlying byte sequence pointers are equal. If the two are equal, then the time complexity
  is O(1), otherwise, the time complexity is O(n), where n is the length of the two strings.


  for the standard Go compiler, in a string value assignment, the destination
  string value and the source string value will share the same underlying byte sequence in
  memory. So the cost of comparing the two strings becomes very small.


  ```go
    bs := make([]byte, 1<<26)
    s0 := string(bs)
    s1 := string(bs)
    s2 := s1


    // s0, s1 and s2 are three equal strings.
    // The underlying bytes of s0 is a copy of bs.
    // The underlying bytes of s1 is also a copy of bs.
    // The underlying bytes of s0 and s1 are two
    // different copies of bs.
    // s2 shares the same underlying bytes with s1.


    startTime := time.Now()
    _ = s0 == s1
    duration := time.Now().Sub(startTime)
    fmt.Println("duration for (s0 == s1):", duration)


    startTime = time.Now()
    _ = s1 == s2
    duration = time.Now().Sub(startTime)
    fmt.Println("duration for (s1 == s2):", duration)


    // output
    // duration for (s0 == s1): 10.462075ms
    // duration for (s1 == s2): 136ns
    // 1ms is 1000000ns! So please try to avoid comparing two long strings if they don't share the same underlying byte sequence.

  ```
