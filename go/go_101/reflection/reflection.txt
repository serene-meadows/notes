-- Reflection Overview
--
  Reflection is a powerful feature in Go that allows a program to inspect,
  and manipulate its own structure and variables at runtime.


  While Go is a statically typed language, reflection provides the "dynamic"
  capabilities needed for high-level libraries like JSON encoders, ORMs,
  and the `fmt` package.





-- The Core Entities: Type and Value
--
  Reflection in Go revolves around two fundamental types defined in the
  `reflect` standard package:


  1. reflect.Type  -> Represents the GO TYPE of a variable (the blueprint).
  2. reflect.Value -> Represents the ACTUAL DATA stored in a variable.


  Every reflection operation begins by taking a standard variable and
  converting it into one of these two reflection objects.





-- The Design Philosophy of Go Reflection
--
  -- Goal: Parity with Non-Reflection Code
  --
    A primary design goal of the Go reflection system is that almost any
    operation you can do in standard code (like adding numbers, calling methods,
    or reading struct fields) should also be possible via reflection.


  -- Reality: Beyond Standard Code
  --
    While 100% parity hasn't been reached, reflection actually allows you to
    do things that are IMPOSSIBLE in standard code, such as:
    - Listing all fields of a struct dynamically.
    - Calling a method by its string name.
    - Creating a new slice or map of a type only known at runtime.





-- Prerequisites: Interfaces are the Key
--
  Reflection is built entirely on top of Go's interface system.
  When you pass a value to a reflection function, it is implicitly boxed
  into an `interface{}` (or `any`).


  Recall that an interface value holds:
  1. The Dynamic Type information.
  2. The Dynamic Value pointer.


  Reflection simply provides a way to "look inside" that interface box.





-- Inspecting Types with reflect.Type
--
  In Go reflection, a type is represented by the `reflect.Type` interface.
  It provides a vast array of methods to inspect the metadata of any Go type,
  such as its name, its underlying "Kind" (e.g., struct, map, int), and
  information specific to that kind (e.g., channel direction or struct fields).


  -- Getting a reflect.Type: The Entry Points
  --
    There are three primary ways to obtain a `reflect.Type` value:

    -- reflect.TypeOf(v any) reflect.Type
    --
      This is the most common way. It returns the type of the value passed to it.

      the interface trap: `TypeOf` accepts an `interface{}`.
      - If you pass a concrete value, it returns the concrete type.
      - If you pass an interface variable, it returns the DYNAMIC TYPE
        (the type of the value currently inside the box).
      - To get the `reflect.Type` of an interface itself using this method, you
        must use a pointer to the interface (e.g., `reflect.TypeOf((*error)(nil)).Elem()`).


    -- reflect.TypeFor[T any]() reflect.Type (Go 1.22+)
    --
      Introduced in Go 1.22, this is the modern, type-safe way to get the
      `reflect.Type` of a type known at compile time. It works perfectly for
      both concrete types and interface types.


    -- Navigational Methods (Elem, Key, Field)
    --
      Once you have a type, you can drill down into its elements (e.g., getting
      the element type of a slice or the key type of a map).





-- Core Methods of reflect.Type
--
  - Kind(): Returns the fundamental category of the type (Pointer, Struct, etc.).
  - Elem(): Returns the element type of a Pointer, Array, Slice, Chan, or Map.
  - Key(): Returns the key type of a Map.
  - ConvertibleTo(u Type): Checks if a value of this type can be converted to type 'u'.
  - Comparable(): Checks if values of this type can be compared with ==.
  - Bits(): Returns the bit size of numeric types.
  - ChanDir(): Returns the direction of a channel type.


  -- Code Demonstration: Deconstructing a Complex Type
  --
    ```go
      func main() {
        // -- Example Type: A receive-only channel of a map

        type A = [16]int16
        var c <-chan map[A][]byte


        // 1. Get the type of the channel
        tc := reflect.TypeOf(c)
        fmt.Println("Outer Kind:    ", tc.Kind())    // Output: chan
        fmt.Println("Chan Direction:", tc.ChanDir()) // Output: <-chan


        // 2. Drill down into the channel element (the Map)
        tm := tc.Elem()
        fmt.Println("Channel Element Kind:", tm.Kind()) // Output: map


        // 3. Drill down into the Map's Key (the Array) and Value (the Slice)
        ta := tm.Key()
        tb := tm.Elem()
        fmt.Printf("Map Structure: Key=%v, Value=%v\n", ta.Kind(), tb.Kind())
        // Output: Map Structure: Key=array, Value=slice


        // 4. Drill down further into the Array and Slice elements
        tx := ta.Elem() // Element of array [16]int16
        ty := tb.Elem() // Element of slice []byte (byte is uint8)


        fmt.Printf("Base Elements: ArrayElem=%v, SliceElem=%v\n", tx.Kind(), ty.Kind())
        // Output: Base Elements: ArrayElem=int16, SliceElem=uint8


        fmt.Printf("Bit Sizes:     int16=%v bits, byte=%v bits\n", tx.Bits(), ty.Bits())
        // Output: Bit Sizes:     int16=16 bits, byte=8 bits


        // -- Property Checks

        // Can we convert an int16 to a uint8?
        fmt.Println("int16 convertible to uint8?", tx.ConvertibleTo(ty)) // Output: true


        // Are slices comparable? (No)
        fmt.Println("Is []byte comparable?       ", tb.Comparable())    // Output: false


        // Are maps comparable? (No)
        fmt.Println("Is map comparable?          ", tm.Comparable())    // Output: false


        // Are arrays comparable? (Yes, if element type is)
        fmt.Println("Is [16]int16 comparable?    ", ta.Comparable())    // Output: true
      }

    ```

    EXPECTED OUTPUT:
    Outer Kind:     chan
    Chan Direction: <-chan
    Channel Element Kind: map
    Map Structure: Key=array, Value=slice
    Base Elements: ArrayElem=int16, SliceElem=uint8
    Bit Sizes:     int16=16 bits, byte=8 bits
    int16 convertible to uint8? true
    Is []byte comparable?        false
    Is map comparable?           false
    Is [16]int16 comparable?     true





-- Handling Kind-Specific Panics (Information Gain)
--
  WARNING: Many methods on `reflect.Type` are kind-specific.
  - Calling `ChanDir()` on a struct type will cause a runtime PANIC.
  - Calling `Key()` on a slice type will cause a runtime PANIC.

  ALWAYS check `Kind()` before calling kind-specific methods in generic code.


  example
    ```go
      func safeInspection(t reflect.Type) {
        if t.Kind() == reflect.Chan {
          fmt.Println("Safe to check direction:", t.ChanDir())
        }
      }

    ```





-- reflect.Type - Elem, Interface Discovery & Implementations
--
  Go's reflection system categorizes every possible type into one of 26
  specific "Kinds" (reflect.Kind). These include basic types (int, string, bool),
  composite types (struct, ptr, slice, map, chan, func), and the interface type.


  -- The Versatility of the .Elem() Method
  --
    The `Elem()` method is one of the most used tools in `reflect.Type`. It is
    used to "unwrap" a type to see what is inside or underneath it.

    -- It works on:
    --
      1. Pointers: Returns the base type (what the pointer points to).
      2. Arrays:   Returns the element type.
      3. Slices:   Returns the element type.
      4. Maps:     Returns the element (value) type.
      5. Channels: Returns the element type.


    Rule: Calling `Elem()` on any other Kind (like an int or a struct)
    will cause a runtime PANIC.


  -- Capturing Interface Types (The Indirection Trick)
  --
    A common hurdle in Go reflection is getting a `reflect.Type` that represents
    an interface itself, rather than the value inside it.


    -- The Problem
    --
      If you call `reflect.TypeOf(myInterface)`, Go boxes the interface.
      `TypeOf` then returns the DYNAMIC type of the value inside. If the
      interface is nil, it returns nil.

    -- The Solution (Indirection)
    --
      1. Create a pointer to the interface: `new(InterfaceType)`.
      2. Pass that to `reflect.TypeOf(...)`. This returns the type `*InterfaceType`.
      3. Call `.Elem()` on that pointer type to get the `InterfaceType` itself.

      Note: In Go 1.22+, `reflect.TypeFor[InterfaceType]()` is the preferred way.


  -- Checking Implementation: .Implements()
  --
    The `Implements(u reflect.Type)` method allows you to check at runtime if a
    type satisfies a specific interface.

    - The receiver can be any type.
    - The argument `u` MUST be an interface type.
    - It returns `true` if the type implements the interface.


  -- Code Demonstration
  --
    ```go
      // Define a custom interface and a type that implements it
      type MyInterface interface {
        m()
      }


      type T []interface{ m() }


      func (T) m() {}


      func main() {
        // -- Step 1: Getting Types

        // tp represents *interface{} (Pointer to a blank interface)
        tp := reflect.TypeOf(new(interface{}))


        // tt represents the defined type T (which is a slice)
        tt := reflect.TypeOf(T{})


        fmt.Println("Initial Kinds: ", tp.Kind(), "|", tt.Kind())
        // Initial Kinds:  ptr | slice


        // -- Step 2: Indirect Interface Discovery

        // ti becomes the reflect.Type for 'interface{}' (blank interface)
        ti := tp.Elem()


        // tim becomes the reflect.Type for 'interface{ m() }' (the slice's element type)
        tim := tt.Elem()


        fmt.Println("Unwrapped Kinds:", ti.Kind(), "|", tim.Kind())
        // Unwrapped Kinds: interface | interface


        // -- Step 3: Implementation Logic Checks --

        // Does type T implement MyInterface? Yes.
        fmt.Println("T implements MyInterface?      ", tt.Implements(tim))
        // T implements MyInterface?       true


        // Does *interface{} implement MyInterface? No.
        fmt.Println("*interface{} implements MyInterface?", tp.Implements(tim))
        // *interface{} implements MyInterface? false


        // Does MyInterface implement itself? Yes.
        fmt.Println("MyInterface implements itself? ", tim.Implements(tim))
        // MyInterface implements itself?  true


        // -- Step 4: The 'any' (interface{}) Universal Rule

        // Every type in Go implements the blank interface.
        fmt.Println("Does T implement any?          ", tt.Implements(ti))
        fmt.Println("Does MyInterface implement any?", tim.Implements(ti))

        // Does T implement any?           true
        // Does MyInterface implement any? true
      }

    ```





-- Reflective Inspection of Fields, Methods, and Functions
--
  Go's reflection system allows us to "dissect" types at runtime. We can
  inspect the internal structure of structs, the signatures of functions,
  and the method sets of any type.


  -- Inspecting Struct Fields
  --
    When dealing with a `reflect.Type` of kind `reflect.Struct`:
    - NumField(): Returns the total count of fields (including unexported ones).
    - Field(i): Returns a `StructField` object containing the name, type, and tag.
    - PkgPath: A crucial property of a field or method.
       - If the field is EXPORTED, PkgPath is an empty string "".
       - If the field is UNEXPORTED, PkgPath contains the package import path.


  -- Inspecting Function Signatures
  --
    For types of kind `reflect.Func`:
    - NumIn() / NumOut(): Returns the count of input parameters and return values.
    - In(i) / Out(i): Returns the `reflect.Type` of a specific parameter or result.
    - IsVariadic(): Returns true if the final parameter is a `...T` type.


  -- The Method Visibility Rule (Crucial)
  --
    There is a major difference in how reflection sees methods depending on
    whether the type is a concrete type or an interface:

    --  Scenario A: Concrete Types (Non-Interface)
    --
      - NumMethod() returns ONLY the count of EXPORTED methods.
      - MethodByName() cannot find unexported methods.
      - This applies even if you are reflecting within the same package.


    --  Scenario B: Interface Types
    --
      - NumMethod() returns the count of ALL methods (exported and unexported).
      - MethodByName() can find any method defined in the interface.


  -- Code Demonstration
  --
    ```go
      // F is a concrete function type
      type F func(string, int) bool


      // m is UNEXPORTED
      func (f F) m(s string) bool {
        return f(s, 32)
      }


      // M is EXPORTED
      func (f F) M() {}


      // I is an interface
      type I interface {
        m(s string) bool // unexported
        M()               // exported
      }


      func main() {
        var x struct {
          F F // Field 0: Exported
          i I // Field 1: Unexported (lowercase i)
        }


        tx := reflect.TypeOf(x)
        fmt.Println("Kind of x:     ", tx.Kind())     // struct
        fmt.Println("Number of fields:", tx.NumField()) // 2


        // -- Field Inspection

        f0 := tx.Field(0)
        f1 := tx.Field(1)
        fmt.Printf("Field 0: Name=%s, PkgPath=%q\n", f0.Name, f0.PkgPath) // Name=F, PkgPath=""
        fmt.Printf("Field 1: Name=%s, PkgPath=%q\n", f1.Name, f1.PkgPath) // Name=i, PkgPath="main"


        // -- Function Signature Inspection
        tf := f0.Type // The type of field F (which is a func)
        fmt.Println("\nFunction Signature (F):")
        fmt.Printf(" Params: %d, Results: %d\n", tf.NumIn(), tf.NumOut())
        fmt.Printf(" Param 0: %v, Param 1: %v, Result 0: %v\n", tf.In(0), tf.In(1), tf.Out(0))


        // -- Method Set Disparity
        ti := f1.Type // The type of field i (which is an interface)


        fmt.Println("\nMethod Counts:")
        // tf (Concrete) only shows 1 method (M). 'm' is unexported and hidden.
        fmt.Printf(" Concrete F methods: %d\n", tf.NumMethod())


        // ti (Interface) shows both methods (m, M).
        fmt.Printf(" Interface I methods: %d\n", ti.NumMethod())


        // Attempting to find 'm' by name
        _, hasInConcrete := tf.MethodByName("m")
        _, hasInInterface := ti.MethodByName("m")
        fmt.Printf(" Found 'm' in Concrete? %t\n", hasInConcrete)
        fmt.Printf(" Found 'm' in Interface? %t\n", hasInInterface)
      }

    ```

      // EXPECTED OUTPUT:
      // Kind of x:      struct
      // Number of fields: 2
      // Field 0: Name=F, PkgPath=""
      // Field 1: Name=i, PkgPath="main"

      // Function Signature (F):
      //  Params: 2, Results: 1
      //  Param 0: string, Param 1: int, Result 0: bool

      // Method Counts:
      //  Concrete F methods: 1
      //  Interface I methods: 2
      //  Found 'm' in Concrete? false
      //  Found 'm' in Interface? true





-- Inspecting Struct Field Tags
--
  Struct tags are metadata strings attached to struct fields. While they are
  technically just strings, they follow a convention of `key:"value"` pairs.
  Reflection allows us to parse and use these tags at runtime.


  -- The reflect.StructTag Type
  --
    When you access a field via reflection (`reflect.Type.Field(i)`), the `Tag`
    property is of type `reflect.StructTag`. This is a specialized string type
    with built-in methods for parsing key-value pairs.


    There are two primary methods for inspection:
    1. Get(key string) string:
       Returns the value associated with the key. If the key is not found,
       it returns an empty string "".
    2. Lookup(key string) (value string, ok bool):
       Returns the value and a boolean indicating if the key was found.
       This is preferred if you need to distinguish between a missing key
       and a key explicitly set to an empty string.


  -- Strict Formatting Rules
  --
    For the `reflect` package to successfully parse your tags, you must follow
    these strict syntactical rules:


    -- Rule 1: No Spaces in the Key/Semicolon Area
    --
      A valid pair must be `key:"value"`.
      - INVALID: `key: "value"` (Space after colon breaks the parser).
      - INVALID: `key name:"value"` (Space in key is illegal).


    -- Rule 2: Values are Literal
    --
      Leading or trailing spaces inside the quotes are treated as part of the value.
      - `json:"id"` vs `json:" id"` are different tags.


    -- Rule 3: Single Line Convention
    --
      Although technically a string, tags should be written on a single line.
      Standard library parsers generally expect this format.


  -- Code Demonstration
  --
    ```go
      type User struct {
        // Multiple keys are separated by spaces
        X int  `max:"99" min:"0" default:"0"`

        // Tags applied to multiple fields on one line apply to both
        Y, Z bool `optional:"yes"`
      }


      func main() {
        uType := reflect.TypeOf(User{})

        // -- Inspecting Field X
        fieldX := uType.Field(0)
        tagX := fieldX.Tag


        fmt.Printf("Tag Type: %T\n", tagX) // Output: reflect.StructTag


        // Lookup returns (value, exists)
        maxVal, exists := tagX.Lookup("max")
        fmt.Printf("Key 'max': value=%s, present=%t\n", maxVal, exists)


        // Get returns just value (empty if missing)
        minVal := tagX.Get("min")
        fmt.Printf("Key 'min': value=%s\n", minVal)


        // Attempting to find a key that doesn't exist on X
        _, xOptionalExists := tagX.Lookup("optional")
        fmt.Printf("Key 'optional' exists on X? %t\n", xOptionalExists)


        // -- Inspecting Field Y and Z
        tagY := uType.Field(1).Tag
        tagZ := uType.Field(2).Tag


        valY, _ := tagY.Lookup("optional")
        valZ, _ := tagZ.Lookup("optional")


        fmt.Printf("Key 'optional' on Y: %s\n", valY)
        fmt.Printf("Key 'optional' on Z: %s\n", valZ)
      }

    ```
      // Output
      // Tag Type: reflect.StructTag
      // Key 'max': value=99, present=true
      // Key 'min': value=0
      // Key 'optional' exists on X? false
      // Key 'optional' on Y: yes
      // Key 'optional' on Z: yes


  -- Common Pitfalls (Information Gain)
  --
    1. Unexported Fields:
       `reflect.TypeOf` can see the tags of unexported fields, but many external
       libraries (like `encoding/json`) use reflection from outside your
       package and will ignore tags on unexported fields.

    2. Comma Conventions:
       Many libraries use commas within the value (e.g., `json:"name,omitempty"`).
       The `reflect` package does NOT parse these commas; it treats the entire
       string "name,omitempty" as the value. The library using the tag is
       responsible for splitting the string by commas.





-- Dynamically Creating Types with reflect
--
  Go's reflection system is not limited to inspecting existing types. It can
  also be used to "mint" brand-new, unnamed composite types at runtime.
  This is essential for frameworks that generate data structures dynamically,
  such as custom serializers or database mappers.


  -- The Type Creator Functions
  --
    The `reflect` package provides a set of "Of" functions to construct types
    from their constituent parts.


    - reflect.ArrayOf(len, elemType): Creates an array type.
    - reflect.ChanOf(dir, elemType): Creates a channel type (Send, Recv, or Both).
    - reflect.PointerTo(baseType): Creates a pointer type. (Note: PtrTo is the legacy name).
    - reflect.SliceOf(elemType): Creates a slice type.
    - reflect.MapOf(keyType, valType): Creates a map type.
    - reflect.FuncOf(ins, outs, variadic): Creates a function signature.
    - reflect.StructOf(fields): Creates a struct type from a slice of StructField.


  -- Code Demonstration: Dynamic Type Construction
  --
    ```go
      func main() {
        // -- Arrays and Chiannels

        // Create [5]int
        ta := reflect.ArrayOf(5, reflect.TypeOf(0))
        fmt.Println("Array Type:  ", ta) // Output: [5]int


        // Create chan<- [5]int (Send-only channel of the array above)
        tc := reflect.ChanOf(reflect.SendDir, ta)
        fmt.Println("Chan Type:   ", tc) // Output: chan<- [5]int


        // -- Pointers and Slices

        // Create *[5]int
        tp := reflect.PointerTo(ta)
        fmt.Println("Pointer Type:", tp) // Output: *[5]int


        // Create []* [5]int (Slice of pointers to the array)
        ts := reflect.SliceOf(tp)
        fmt.Println("Slice Type:  ", ts) // Output: []*[5]int


        // -- Maps and Functions

        // Create map[[5]int]chan<- [5]int
        // Note: The key type MUST be comparable. Arrays are comparable if elements are.
        tm := reflect.MapOf(ta, tc)
        fmt.Println("Map Type:    ", tm) // Output: map[[5]int]chan<- [5]int


        // Create a function signature: func([5]int) (*[5]int, chan<- [5]int)
        ins := []reflect.Type{ta}
        outs := []reflect.Type{tp, tc}
        tf := reflect.FuncOf(ins, outs, false) // false = not variadic
        fmt.Println("Func Type:   ", tf) // Output: func([5]int) (*[5]int, chan<- [5]int)


        // -- Structs

        // Create struct { Age string }
        fields := []reflect.StructField{
          {
            Name: "Age",
            Type: reflect.TypeOf(""),
          },
        }
        tt := reflect.StructOf(fields)
        fmt.Println("Struct Type: ", tt)            // Output: struct { Age string }
        fmt.Println("Field Count: ", tt.NumField()) // Output: 1
      }

    ```
      // EXPECTED OUTPUT:
      // Array Type:   [5]int
      // Chan Type:    chan<- [5]int
      // Pointer Type: *[5]int
      // Slice Type:   []*[5]int
      // Map Type:     map[[5]int]chan<- [5]int
      // Func Type:    func([5]int) (*[5]int, chan<- [5]int)
      // Struct Type:  struct { Age string }
      // Field Count:  1


  -- Limitations and Pitfalls (Go 1.25)
  --
    While powerful, the reflection type-builder has specific limitations:

    -- No Dynamic Interfaces
    --
      Up to Go 1.25, there is NO way to create a new interface type at runtime via
      reflection. Interfaces must be defined statically in your source code.


    -- Struct Embedding Pitfalls
    --
      You can create a struct with embedded fields by setting the `Anonymous`
      field to `true` in a `reflect.StructField`. However, this is dangerous:
      1. Complexity: Handling promoted methods and fields correctly is difficult.
      2. Unexported Types: Embedding an unexported type from another package
         is generally impossible or leads to panics.
      3. Performance: `reflect.StructOf` is a relatively expensive operation.


    -- Map Key Constraint
    --
      When using `reflect.MapOf`, if you pass a key type that is not comparable
      (like a slice), the function will PANIC immediately.





-- Data Manipulation with reflect.Value
--
  While reflect.Type represents the "blueprint" of a type, reflect.Value
  represents the actual "live data" stored in a variable.


  Reflection allows you to read, track, and—most importantly—modify the
  underlying value of a variable at runtime.


  --  Getting a reflect.Value
  --
    - reflect.ValueOf(v any) reflect.Value

    The Interface Rule: Just like TypeOf, ValueOf accepts an interface{}.
    1. If you pass a concrete value, it returns a reflect.Value for that data.
    2. If you pass an interface variable, it returns a reflect.Value representing
       the dynamic value (what is inside the box).


  -- Settability and Addressability
  --
    Before you can modify a value via reflection, two conditions must be met:

    -- CanAddr() (Addressability)
    --
      Returns true if the value has a memory address that reflection can reach.
      A value returned directly by `reflect.ValueOf(x)` is always a copy, and
      therefore is NOT addressable.


    -- CanSet() (Modifiability)
    --
      Returns true if the value is both addressable AND was not obtained by
      accessing unexported struct fields.

      rule: To modify a variable 'x', you must pass its address `&x` to
      ValueOf, and then call `.Elem()` to "dereference" the reflection object.


  -- Code Demonstration: Modifying a Basic Value
  --
    ```go
    func modifyBasicValue() {
      n := 123

      // vp represents the pointer (*int).
      // Values returned by ValueOf are always read-only/un-settable initially.
      vp := reflect.ValueOf(&n)
      fmt.Println("Pointer CanSet:", vp.CanSet()) // Output: false


      // .Elem() "dereferences" the reflection pointer.
      // It gives us a reflect.Value representing the actual integer 'n'.
      vn := vp.Elem()


      fmt.Println("Unwrapped CanSet:", vn.CanSet()) // Output: true
      fmt.Println("Unwrapped CanAddr:", vn.CanAddr()) // Output: true


      // Modify the value. We can use Set() with a reflect.Value
      // or kind-specific methods like SetInt().
      vn.SetInt(789)


      fmt.Println("Updated n:", n) // Output: 789
    }

    ```


  -- Navigating Pointers: Elem() vs. Indirect()
  --
    When you have a reflect.Value representing a pointer, you have two ways
    to reach the value it points to:

    1. v.Elem():
       - If v is a Ptr: Returns the value it points to.
       - If v is an Interface: Returns the dynamic value inside.
       - Otherwise: PANICS.


    2. reflect.Indirect(v):
       - If v is a Ptr: Returns the value it points to (same as Elem).
       - Otherwise: Returns 'v' itself (no-op). This is "safer" as it won't panic.


  -- Struct Field Visibility and Modification
  --
    Reflection respects Go's visibility rules.
    - You CAN access unexported fields (lowercase) using .Field().
    - You CANNOT modify (Set) unexported fields.
    - Attempting to Set an unexported field will result in a runtime PANIC.


    example
      ```go
        func structReflectExample() {
          var s struct {
            X int // Exported
            y int // Unexported
          }


          // Get addressable struct via pointer + Elem/Indirect
          vs := reflect.Indirect(reflect.ValueOf(&s))


          vx := vs.Field(0) // Field X
          vy := vs.Field(1) // Field y


          // Checking modifiability
          fmt.Printf("X (Exported) - CanSet: %t, CanAddr: %t\n", vx.CanSet(), vx.CanAddr())   // X (Exported) - CanSet: true, CanAddr: true
          fmt.Printf("y (Private)  - CanSet: %t, CanAddr: %t\n", vy.CanSet(), vy.CanAddr())   // y (Private)  - CanSet: false, CanAddr: true

          // Setting X works
          vx.SetInt(100)


          // Setting y would PANIC:
          // vy.SetInt(200) // panic: reflect: reflect.Value.SetInt using value obtained using unexported field


          fmt.Println("Struct state:", s) // Output: {100 0}
        }

      ```





-- Advanced Manipulation with reflect.Value
--
  Beyond simple inspection and setting, Go's reflection system allows for
  deep unwrapping of interfaces, dynamic function creation, and the
  programmatic invocation of methods and functions.


  -- Deep Unwrapping with .Elem()
  --
    The `.Elem()` method is highly versatile. It is used to:
    1. Dereference a Pointer: Get the value the pointer points to.
    2. Unwrap an Interface: Get the dynamic value stored inside the interface box.

    By chaining `.Elem()`, you can navigate through multiple layers of pointers
    and interfaces to reach a base value for modification.


  example
    ```go
      func deepUnwrapExample() {
        var z = 123
        var y = &z               // y is *int
        var x interface{} = y    // x is an interface holding *int


        // To modify 'z' starting from 'x':
        // 1. Start with the address of the interface box so it is settable.
        v := reflect.ValueOf(&x) // Type: *interface{}


        // 2. vx = v.Elem()      // Type: interface{} (the content of x)
        vx := v.Elem()


        // 3. vy = vx.Elem()     // Type: *int (the dynamic value inside x)
        vy := vx.Elem()


        // 4. vz = vy.Elem()     // Type: int (the value pointed to by y)
        vz := vy.Elem()


        // Now we can modify the original 'z'
        vz.Set(reflect.ValueOf(789))
        fmt.Println("Value of z after deep set:", z) // Value of z after deep set: 789
      }

    ```





-- Dynamic Function Creation (reflect.MakeFunc)
--
  `reflect.MakeFunc` allows you to define a single generic-style function logic
  and "bind" it to different strongly-typed function variables at runtime.
  This is a powerful way to implement logic that works on any slice type.


  Logic: Take a slice and return a new slice with elements in reverse order.
    ```go
      func InvertSliceLogic(args []reflect.Value) []reflect.Value {
        inSlice := args[0]
        n := inSlice.Len()


        // Create a new slice of the exact same type as the input.
        outSlice := reflect.MakeSlice(inSlice.Type(), 0, n)


        for i := n - 1; i >= 0; i-- {
          element := inSlice.Index(i)
          outSlice = reflect.Append(outSlice, element)
        }


        // Must return a slice of reflect.Value matching function results.
        return []reflect.Value{outSlice}
      }


      // Bind takes a pointer to a function variable and a generic logic function.
      func Bind(p interface{}, f func([]reflect.Value) []reflect.Value) {
        invertFuncVar := reflect.ValueOf(p).Elem()


        // reflect.MakeFunc creates a typed function value from generic logic.
        newFunc := reflect.MakeFunc(invertFuncVar.Type(), f)


        // Assign the dynamically created function to the variable.
        invertFuncVar.Set(newFunc)
      }


      func dynamicFunctionDemo() {
        // We declare function variables but don't define them yet.
        var invertInts func([]int) []int
        var invertStrs func([]string) []string


        // We bind our generic logic to these typed variables.
        Bind(&invertInts, InvertSliceLogic)
        Bind(&invertStrs, InvertSliceLogic)


        fmt.Println("Inverted Ints:", invertInts([]int{2, 3, 5}))      // Inverted Ints: [5 3 2]
        fmt.Println("Inverted Strs:", invertStrs([]string{"Go", "C"})) // Inverted Strs: [C Go]
      }

    ```





-- Programmatic Invocation (.Call)
--
  You can execute a function or method using reflection by calling `.Call()`.
  
  -- Key Rules 
  --
    1. Inputs: Takes a `[]reflect.Value` representing the arguments.
    2. Outputs: Returns a `[]reflect.Value` representing the return results.
    3. Visibility Trap: You CANNOT use values obtained from unexported (private)
       fields as arguments in a Call. Doing so causes a runtime PANIC.


  example
    ```go
      type Calculator struct {
        A int // Exported
        b int // Unexported (Private)
      }


      func (c Calculator) AddSub(n int) (int, int) {
        return n + (c.A + c.b), n + (c.A - c.b)
      }


      func dynamicCallDemo() {
        calc := Calculator{A: 5, b: 2}
        vCalc := reflect.ValueOf(calc)


        // -- Calling a Method by Name 
        vMethod := vCalc.MethodByName("AddSub")


        // Provide arguments as a slice of reflect.Value
        inputs := []reflect.Value{reflect.ValueOf(3)}
        results := vMethod.Call(inputs)


        fmt.Printf("AddSub Results: %d, %d\n", results[0].Int(), results[1].Int())


        // -- Calling an Anonymous Function 

        negate := func(x int) int { return -x }
        vFunc := reflect.ValueOf(negate)


        // Pass the result of the first call into the second call.
        negResults := vFunc.Call(results[:1])
        fmt.Println("Negated result:", negResults[0].Int())


        // -- The Visibility Panic 

        // Using the Exported field 'A' as an argument:
        fmt.Println("Call using A:", vFunc.Call([]reflect.Value{vCalc.FieldByName("A")})[0].Int())


        // DANGER: Using the Unexported field 'b' as an argument:
        // fmt.Println(vFunc.Call([]reflect.Value{vCalc.FieldByName("b")}))
        //
        // RUNTIME PANIC: reflect: reflect.Value.Call using value obtained using unexported field
      }

    ```





-- Reflection on Maps, Channels, and Dynamic Select
--
  Go's reflection system provides specialized methods for manipulating
  complex container types like Maps and Channels. It even allows for
  constructing "Select" statements dynamically at runtime.


  -- Reflection on Maps
  --
    You can use `reflect.Value` to dynamically read, write, and delete entries
    in a map.


    -- Key Methods 
    --
      - v.SetMapIndex(key, value): Sets a value for a key.
        - If 'value' is a zero reflect.Value (created via reflect.Value{}),
          it DELETES the entry for that key.
      - v.MapRange(): Returns an iterator (MapIter) for the map. (Go 1.12+)
      - iter.Next(): Advances the iterator.
      - iter.Key() / iter.Value(): Retrieves the current entry data.


    example
      ```go
        func mapReflectionDemo() {
          m := map[string]int{"Unix": 1973, "Windows": 1985}
          v := reflect.ValueOf(m)


          // -- Modifying and Deleting 

          // Deletion: Pass an empty reflect.Value{} as the second argument.
          v.SetMapIndex(reflect.ValueOf("Windows"), reflect.Value{})


          // Addition/Modification: Pass a valid reflect.Value.
          v.SetMapIndex(reflect.ValueOf("Linux"), reflect.ValueOf(1991))


          // -- Iteration with MapRange --

          fmt.Println("--- Map Contents via Reflection ---")
          iter := v.MapRange()
          for iter.Next() {
            fmt.Printf("%s \t: %d\n", iter.Key(), iter.Value())
          }
        }

      ```





  -- Reflection on Channels
  --
    Reflection allows for both blocking and non-blocking channel operations.


    -- Blocking Methods 
    --
      - v.Send(val): Equivalent to `c <- val`.
      - v.Recv(): Equivalent to `val, ok := <-c`. Returns (value, receivedBeforeClosed).


    -- Non-Blocking Methods (Select Shorthands) 
    --
      - v.TrySend(val): Equivalent to a select block with a default case.
        Returns 'true' if the value was sent, 'false' otherwise.
      - v.TryRecv(): Equivalent to a select block with a default case.
        Returns (value, success).


    example
      ```go
        func channelReflectionDemo() {
          c := make(chan string, 2)
          vc := reflect.ValueOf(c)


          // 1. Synchronous Send
          vc.Send(reflect.ValueOf("C"))


          // 2. Non-blocking Send (TrySend)
          success := vc.TrySend(reflect.ValueOf("Go"))
          fmt.Println("TrySend 'Go' success: ", success) // true (buffer has space)


          success = vc.TrySend(reflect.ValueOf("C++"))
          fmt.Println("TrySend 'C++' success:", success) // false (buffer is full)


          // 3. Inspection
          fmt.Printf("Channel Len: %d, Cap: %d\n", vc.Len(), vc.Cap()) // 2, 2


          // 4. TryRecv (Non-blocking)
          val, ok := vc.TryRecv()
          fmt.Printf("TryRecv: %s, success: %t\n", val.String(), ok) // C, true


          // 5. Recv (Blocking)
          val, ok = vc.Recv()
          fmt.Printf("Recv:    %s, sentBeforeClosed: %t\n", val.String(), ok) // Go, true
        }

      ```





-- Dynamic Select (`reflect.Select`)
--
  The `reflect.Select` function is one of the most advanced features in Go.
  It allows you to perform a "select" operation on a set of channels that
  is only known at runtime.

// -- The SelectCase Struct --
// You provide a slice of `reflect.SelectCase` objects. Each case specifies:
// - Dir: The direction (reflect.SelectSend, SelectRecv, or SelectDefault).
// - Chan: The channel Value.
// - Send: The value to send (if Dir is SelectSend).

// -- The Return Value --
// reflect.Select returns:
// 1. The index of the chosen case.
// 2. The value received (if it was a Recv case).
// 3. A boolean indicating if the receive was sent before the channel closed.

func dynamicSelectDemo() {
	c := make(chan int, 1)
	vc := reflect.ValueOf(c)

	// Fill the channel buffer
	vc.TrySend(reflect.ValueOf(123))

	// Define our cases dynamically
	vSendVal := reflect.ValueOf(789)
	vZero := reflect.Value{} // Used for empty fields

	cases := []reflect.SelectCase{
		{Dir: reflect.SelectDefault, Chan: vZero, Send: vZero}, // Index 0
		{Dir: reflect.SelectRecv,    Chan: vc,    Send: vZero}, // Index 1
		{Dir: reflect.SelectSend,    Chan: vc,    Send: vSendVal}, // Index 2
	}

	// Logic: ch already has 123. Send is blocked. Recv is ready.
	// Default is ready. Go will pick Case 1 (Recv) or Case 0 (Default).
	// In this specific state (buffer full), Case 1 is ready to drain.
	selIndex, vRecv, ok := reflect.Select(cases)

	fmt.Printf("Selected Case Index: %d\n", selIndex)
	if selIndex == 1 {
		fmt.Printf("Received Value:      %d (OK: %t)\n", vRecv.Int(), ok)
	}

	// -------------------------------------------------------------------------
	// -- Handling Closed Channels --
	// -------------------------------------------------------------------------
	vc.Close()

	// We call Select again, but only with the Default and Recv cases.
	// Since channel is closed, Recv will return the zero value and ok=false.
	selIndex, _, ok = reflect.Select(cases[:2])
	fmt.Printf("Woken by Close - Index: %d, sentBeforeClosed: %t\n", selIndex, ok)
}

/*
================================================================================
EXPECTED OUTPUT:
================================================================================
Selected Case Index: 1
Received Value:      123 (OK: true)
Woken by Close - Index: 1, sentBeforeClosed: false
================================================================================
*/


func main() {
	fmt.Println("=== Map Reflection ===")
	mapReflectionDemo()
	fmt.Println("\n=== Channel Reflection ===")
	channelReflectionDemo()
	fmt.Println("\n=== Dynamic Select ===")
	dynamicSelectDemo()
}





// #############################################################################
// #
// #        Go: reflect.Value Validity, Interoperability, and Safety
// #
// #############################################################################

// Working with Go reflection requires understanding the state of a reflect.Value.
// A reflect.Value can represent actual data, a zero value of a type, or it can
// be completely "Invalid" (representing nothing).


// =============================================================================
// ::  1. Invalid reflect.Value (The "Nothing" Value)
// =============================================================================

// Not every `reflect.Value` contains data. Some represent "nothing."
//
// -- How to get an Invalid Value --
// 1. Zero Value: Declaring `var v reflect.Value` without initialization.
// 2. Nil Dereference: Calling `.Elem()` on a `reflect.Value` that represents
//    a nil pointer.
// 3. Nil Interface: Calling `.Elem()` on a `reflect.Value` representing a
//    nil interface.

package main

import (
	"fmt"
	"reflect"
	"time"
)

func validityDemo() {
	// A zero-value reflect.Value is "Invalid"
	var z reflect.Value
	fmt.Printf("Zero reflect.Value:  %v (Valid? %t)\n", z, z.IsValid())

	// Dereferencing a nil pointer results in an Invalid Value
	vPtr := reflect.ValueOf((*int)(nil)).Elem()
	fmt.Printf("Nil Ptr Elem:       %v (Valid? %t)\n", vPtr, vPtr.IsValid())
	fmt.Println("Is zero == nilElem? ", vPtr == z) // true

	// Indexing a slice of interfaces where the element is nil
	sliceWithNil := []interface{}{nil}
	iVal := reflect.ValueOf(sliceWithNil).Index(0)
	fmt.Printf("Interface(nil):     %v (Valid? %t)\n", iVal, iVal.IsValid())

	// BUT, calling .Elem() on that boxed nil interface returns an Invalid Value
	fmt.Printf("Interface(nil).Elem:%v (Valid? %t)\n", iVal.Elem(), iVal.Elem().IsValid())
}

/*
================================================================================
EXPECTED OUTPUT:
================================================================================
Zero reflect.Value:  <invalid reflect.Value> (Valid? false)
Nil Ptr Elem:       <invalid reflect.Value> (Valid? false)
Is zero == nilElem?  true
Interface(nil):     <nil> (Valid? true)
Interface(nil).Elem:<invalid reflect.Value> (Valid? false)
================================================================================
*/


// =============================================================================
// ::  2. Converting reflect.Value back to Go (The .Interface() Method)
// =============================================================================

// The `Interface()` method is the inverse of `reflect.ValueOf()`.
// It extracts the data from the reflection object and packs it into an
// `interface{}` (any).
//
// You then use a standard type assertion to get your concrete Go type back.

func interoperabilityDemo() {
	vx := reflect.ValueOf(123)
	vy := reflect.ValueOf("abc")
	vz := reflect.ValueOf([]bool{false, true})
	vt := reflect.ValueOf(time.Time{})

	// 1. Convert reflect.Value -> interface{}
	// 2. Type Assert interface{} -> Concrete Type
	x := vx.Interface().(int)
	y := vy.Interface().(string)
	z := vz.Interface().([]bool)

	// You can even extract function values from methods
	isZeroFunc := vt.MethodByName("IsZero").Interface().(func() bool)

	fmt.Println("Extracted:", x, y, z, isZeroFunc())
}

/*
================================================================================
EXPECTED OUTPUT:
================================================================================
Extracted: 123 abc [false true] true
================================================================================
*/


// =============================================================================
// ::  3. The "Unexported Field" Panic
// =============================================================================

// **CRITICAL SAFETY WARNING**:
// While reflection allows you to READ unexported (private) fields using
// `.Field(i)`, you are strictly FORBIDDEN from calling `.Interface()` on
// those values.
//
// Doing so violates Go's encapsulation rules and triggers an unrecoverable
// runtime PANIC.

func visibilityPanicDemo() {
	type T struct {
		Public  int
		private int
	}
	t := T{Public: 1, private: 2}
	v := reflect.ValueOf(t)

	// Field(0) is "Public"
	fmt.Println("Public field Interface():", v.Field(0).Interface()) // OK

	// Field(1) is "private"
	fmt.Println("Private field via reflection:", v.Field(1)) // OK to print/inspect

	// DANGER:
	/*
	fmt.Println(v.Field(1).Interface())
	// RUNTIME PANIC: reflect.Value.Interface: cannot return value obtained from unexported field
	*/
}


// =============================================================================
// ::  4. Checking for Zero Values (IsZero)
// =============================================================================

// Introduced in Go 1.13, `v.IsZero()` checks if the underlying value is the
// default zero value for its type (e.g., 0 for int, "" for string, nil for ptr).
//
// **Distinction**:
// - `IsValid()`: Checks if the reflect.Value itself exists/is not the zero Value.
// - `IsZero()`: Checks if the DATA represented is the type's zero value.

func isZeroDemo() {
	v1 := reflect.ValueOf(0)
	v2 := reflect.ValueOf(42)

	fmt.Printf("Is 0 a zero value?  %t\n", v1.IsZero())  // true
	fmt.Printf("Is 42 a zero value? %t\n", v2.IsZero()) // false
}

/*
================================================================================
EXPECTED OUTPUT:
================================================================================
Is 0 a zero value?  true
Is 42 a zero value? false
================================================================================
*/

func main() {
	fmt.Println("--- Validity Demo ---")
	validityDemo()
	fmt.Println("\n--- Interoperability Demo ---")
	interoperabilityDemo()
	fmt.Println("\n--- Visibility Rules ---")
	visibilityPanicDemo()
	fmt.Println("\n--- IsZero Demo ---")
	isZeroDemo()
}





// #############################################################################
// #
// #        Go: Safe Conversions with reflect.Value.CanConvert
// #
// #############################################################################

// Go 1.17 introduced a powerful feature: the ability to convert a slice into 
// a pointer to an array (e.g., converting []int to *[5]int). 
//
// However, this operation is risky. If the slice's length is smaller than the 
// array's length, the program will PANIC. To handle this safely at runtime 
// using reflection, we use two distinct methods: 
// 1. .ConvertibleTo() (Type level)
// 2. .CanConvert()    (Value level)


// =============================================================================
// ::  1. Type vs. Value Checks
// =============================================================================

// -----------------------------------------------------------------------------
// -- 1.1. reflect.Type.ConvertibleTo(T reflect.Type) --
// -----------------------------------------------------------------------------
// This checks if a value of the source type could POSSIBLY be converted to 
// type T. 
// - Since Go allows slice-to-array-pointer conversions, this returns 'true' 
//   for any slice and any array pointer of the same element type, regardless 
//   of the actual lengths involved.

// -----------------------------------------------------------------------------
// -- 1.2. reflect.Value.CanConvert(T reflect.Type) --
// -----------------------------------------------------------------------------
// This checks if the SPECIFIC value (the actual data and its current length) 
// can be safely converted to type T without panicking.
// - For slices, this checks if `len(slice) >= len(array)`.


// =============================================================================
// ::  2. Code Demonstration
// =============================================================================

package main

import (
	"fmt"
	"reflect"
)

func main() {
	// We have a slice of length 5
	s := reflect.ValueOf([]int{1, 2, 3, 4, 5})
	ts := s.Type()

	// Define two target types:
	t1 := reflect.TypeOf(&[5]int{}) // Pointer to array of length 5
	t2 := reflect.TypeOf(&[6]int{}) // Pointer to array of length 6

	// -------------------------------------------------------------------------
	// --  Inspection 1: ConvertibleTo (Type Level) --
	// -------------------------------------------------------------------------
	// Both are true because a slice of int is syntactically allowed to be 
	// converted to a pointer of an int array of any size.
	fmt.Println("Type []int convertible to *[5]int?", ts.ConvertibleTo(t1)) // true
	fmt.Println("Type []int convertible to *[6]int?", ts.ConvertibleTo(t2)) // true

	// -------------------------------------------------------------------------
	// --  Inspection 2: CanConvert (Value Level) --
	// -------------------------------------------------------------------------
	
	// True: The slice has 5 elements, so it can fit into a *[5]int.
	fmt.Println("Value (len 5) can convert to *[5]int?", s.CanConvert(t1)) // true

	// False: The slice only has 5 elements. Trying to convert it to a *[6]int 
	// would require 6 elements. Doing this in standard Go would PANIC.
	// reflect.Value.CanConvert tells us this is unsafe.
	fmt.Println("Value (len 5) can convert to *[6]int?", s.CanConvert(t2)) // false

	// -------------------------------------------------------------------------
	// -- Performing the Conversion --
	// -------------------------------------------------------------------------
	if s.CanConvert(t1) {
		convertedValue := s.Convert(t1)
		fmt.Printf("Conversion Result: %v, Type: %T\n", convertedValue, convertedValue.Interface())
	}
}

/*
================================================================================
EXPECTED OUTPUT:
================================================================================
Type []int convertible to *[5]int? true
Type []int convertible to *[6]int? true
Value (len 5) can convert to *[5]int? true
Value (len 5) can convert to *[6]int? false
Conversion Result: &[1 2 3 4 5], Type: *[5]int
================================================================================
*/


// =============================================================================
// ::  3. Summary
// =============================================================================

// 1. Use `Type.ConvertibleTo` to check if the logic of your code is sound 
//    regarding Go's type system.
// 2. Use `Value.CanConvert` to guard against runtime panics when dealing with 
//    dynamic data like slices, where the type is compatible but the 
//    content (length) might not be.
