Go has only one looping keyword: `for`. However, its flexible syntax allows it
to behave like the `for`, `while`, and infinite loops found in other languages.





-- c style loop
--
  The most complete form of the `for` loop consists of three components
  separated by semicolons.

  //   for initialization; condition; post {
  //       // loop body
  //   }


  - initialization: A short statement (e.g., `i := 0`) that runs exactly
    once before the loop begins. Variables declared here are scoped only to
    the `for` loop.

  - condition: An expression evaluated before each iteration. If it is `true`,
    the loop body executes. If `false`, the loop terminates.

  - post: A short statement (e.g., `i++`) that runs at the end of each
    iteration


  Key Syntax Rule
    Unlike C, Java, or C++, the three components are NOT
    enclosed in parentheses `()`.


  example
    ```go
      func cStyleLoop() {
        for i := 0; i < 5; i++ {
          fmt.Println("C-style:", i)
        }
      }

    ```





-- Optional Components and Common Forms
--
  All three components of the `for` loop are optional. This flexibility gives
  rise to its different common forms.


  -- The Condition-Only (`while`) Form
  --
    If you omit the `initialization` and `post` statements (and their semicolons),
    you get a loop that behaves exactly like a `while` loop from other languages.

    example
      ```go
        func whileStyleLoop() {
          n := 0
          // This loop runs as long as `n < 5` is true.
          for n < 5 {
            fmt.Println("While-style:", n)
            n++
          }


          // This is the more verbose but equivalent form:
          // for ; n < 10; { ... }
        }

      ```


  -- The Infinite Loop Form
  --
    If the `condition` part is also omitted, it defaults to `true`, creating an
    infinite loop. This form is very common in Go for servers or background tasks
    that are meant to run forever or be stopped by an explicit `break`, `return`,
    or `panic`.


    ```go
      func infiniteLoop() {
        // The `for {}` form is the most idiomatic and common way to write this.
        for {
          fmt.Println("This would run forever...")
          break // Exit the loop immediately.
        }


        // The following forms are all completely equivalent:
        // for ; ; { }
        // for true { }
        // for ; true; { }
      }

    ```


  -- Omitting Only the `condition`
  --
    You can also omit just the `condition`, which, again, defaults to `true`.

    example
      ```go
        func noConditionLoop() {
          for i := 0; ; i++ { // Equivalent to `for i := 0; true; i++`
            if i >= 5 {
              break
            }
            fmt.Println("No-condition style:", i)
          }
        }

      ```





-- Variable Shadowing within the Loop Body
--
  It is possible to declare a new variable inside the loop's body with the same
  name as the loop's control variable. This is called "shadowing".
  The inner variable "hides" the outer one for the duration of its scope.

  ```go
    func shadowingExample() {
      // The loop variable `i` is declared here.
      for i := 0; i < 3; i++ {
        fmt.Printf("Outer loop variable 'i' = %d\n", i)

        // This line declares a NEW variable, also named `i`.
        // It is initialized with the current value of the outer loop `i`.
        // From this point until the end of the block, `i` refers to this new variable.
        i := i

        // We modify the inner, shadowed `i`.
        // This has NO effect on the outer loop's control variable.
        i = 100
        _ = i // Use the inner `i` to satisfy the compiler.
      }
      // After this loop, you would see:
      // Outer loop variable 'i' = 0
      // Outer loop variable 'i' = 1
      // Outer loop variable 'i' = 2
    }

  ```
