In Go, importing a package gives your current file access to the exported
elements of that package.


-- Key Rules
--
  1. File-Scoped: An import declaration only applies to the specific source 
     file it is written in, NOT the entire package.
  2. Exported Elements Only: You can only access elements (functions, types, 
     variables) from the imported package that start with a Capital Letter.





-- Basic & Grouped Imports
--
  While you can import packages one by one...  
  // import "fmt"
  // import "math/rand"


  ...it is highly idiomatic to group them using parentheses `()`
  Standard library packages are typically grouped separately from third-party ones.


  example
    ```go
      import (
        "fmt"
        "math/rand"
      )


      func basicImportExample() {
        // We use the package name as the prefix to access its exported elements.
        // `Uint32` is exported (capitalized). `fmt.Printf` is exported.
        fmt.Printf("Random number: %v\n", rand.Uint32())
      }

    ```





-- Aliased Imports (Renaming)
--
  -- Syntax: import alias "path/to/package"
  --
    You can provide a custom name (alias) for an imported package. 
    This is incredibly useful for:
    1. Resolving name collisions (e.g., two packages named "log").
    2. Shortening very long package names.


    example
      ```go
        import (
          format "fmt"        // "fmt" is now accessed via the 'format' prefix
          random "math/rand"  // "math/rand" is now accessed via the 'random' prefix
        )


        func aliasExample() {
          format.Print("A random number: ", random.Uint32(), "\n")

          // The following line would FAIL to compile, because the default names
          // "fmt" and "rand" are no longer recognized in this file.
          // fmt.Print("A random number: ", rand.Uint32(), "\n")
        }

      ```





-- Dot Imports (.)
--
  A dot import allows you to access all exported elements of the imported 
  package directly, WITHOUT using the package prefix.


  WARNING: Dot imports are generally discouraged in production code because
  they pollute the local namespace and make it hard to tell where a function
  came from. They are mostly used in testing files.


  example
    ```go
      import (
        . "fmt"
        . "time"
      )

      func dotImportExample() {
        // Look closely: No "fmt." or "time." prefixes!
        Println("Current time:", Now()) 
      }

    ```





-- Blank / Anonymous Imports (_)
--
  -- Syntax: 
  --
    import _ "path/to/package"


  A blank import imports the package purely for its SIDE EFFECTS.
  You cannot access any exported elements from an anonymously imported package.


  -- Why do this?
  --
    When a package is imported, Go automatically runs all `init()` functions 
    defined within that package. Blank imports are commonly used to register
    database drivers (like Postgres/MySQL) or start background profiling tools


  example
    ```go
      import _ "net/http/pprof" 

      func blankImportExample() {
        // The `init()` functions inside "net/http/pprof" have now run, 
        // setting up profiling HTTP endpoints in the background automatically.
        
        // However, we cannot call any functions from `pprof` directly here.
      }

    ```





-- the Strict "Unused Import" Rule
--
  Go compiler is notoriously strict: Every non-blank import MUST be used at 
  least once in the file. If you import a package and don't use it, your
  code WILL NOT COMPILE.


  This keeps Go binaries small and compilation times lightning fast by
  preventing dead dependencies from accumulating.


  example
    ```go
      import (
        "net/http" // COMPILE ERROR: imported and not used
        . "time"   // COMPILE ERROR: imported and not used
      )

    ```


  example
    ```go
      import (
        myFmt "fmt"   // OKAY: We use `myFmt` below.
        _ "math/rand" // OKAY: Blank imports are exempt from the "must use" rule.
      )


      func unusedImportExample() {
        myFmt.Println("Because I used myFmt here, the compiler is happy.")
      }

    ```
